git的使用方法

发表于 2019-08-08 | 分类于 内核工具
字数统计: 745 | 阅读时长 ≈ 4

1.创建fork与upstream 保持同步更新

  • git init 初始化一个仓库
    [root@localhost openvswitch]# git init

  • 添加自己的remote repo和upstream的repo

  • sync source code from upstream
    [root@localhost openvswitch]# git fetch upstream

  • checkout the master branch
    [root@localhost openvswitch]# git checkout master

  • merge the upstream source code to my master
    [root@localhost openvswitch]# git merge upstream/master

  • push the upstream source code to my remote repo
    [root@localhost openvswitch]# git push -u origin master

2.git some of operation

  • 删除 local/remote branch

  • 保存github的username和passwd

  • clone remote branch without fetch other branchs

  • checkout remote branch to local branch

    stackoverflow links

  • push local branch to remote branch

  • 远端仓库重命名

    • Rename the local branch to the new name

    • Delete the old branch on remote - where <remote> is, for example, origin
      git push OVS --delete old_name

    • Push the new branch to remote
      git push OVS new_name

    • Reset the upstream branch for the new_name local branch
      git push OVS-u new-name

3.git 提交的使用方法

  • 添加文件夹和文件到远端repo中

    1. 添加文件git add --all Part02/
    2. git commit -m “xxxxxxxxxxxxxxxxxxx”
    3. git push -u origin master

  • 修改已经commit的内容

    • 还没有push的commit
      git commit --amend
      git push -u origin master
    • 已经push的commit的修改
      git commit --amend
      git push -u origin master --force

  • 修改指定的commit的内容
    1.显示commit的信息，选定需要修改的commit的hash id,例如标红的
    # git log --oneline

    2.选定标红之前的commit id号 rebase,选择需要修改的那行，将 pick 替换成 edit 之后保存
    # git rebase -i 1333d0a

    3.修改commit的内容
    a.修改作者名和email
    # git commit --amend --author="Author Name <email@address.com>"
    b.修改提交时间
    # git commit --amend --date="$(date -R)"
    c.commit 的message
    # git commit --amend -m “xxxxxx”

    4.完成rebase 以及提交之前已经存在的message信息
    # git rebase --continue
    5.push到remote
    # git push origin master --force // git pull && git push origin master

字数统计: 1.3k | 阅读时长 ≈ 6

安装sar工具及enable

  • 安装sar工具

    yum install sysstat

  • 修改配置打开sar

  • 查看crontab的内容

使用sar工具

  • 查看cpu使用率

    sar -u

  • 看平均负载情况

    sar -q

  • 查看内存负载

    sar -r

  • 查看内存页面交换情况

    sar -W

  • 查看内存的情况

    sar -B

  • 查看IO的情况

    sar -b
